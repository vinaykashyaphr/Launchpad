Data Conversion:

Instructions on how to use the data conversion tool.

Arecibo:

Select the file to be QAed and enter its job number. Next choose which type of document it is (TR, SB, CMM, etc.). Indicate whether its D number is its publication number. Next, select which checks to run on the document:

Automated QA: Uses job specific data obtained from Jira (I.e., Publication number, ECCN, etc.) to check front matter accuracy of the document.

LEP Check: Checks the pages found in the List of Effective Pages (LEP) against the pages in the body of the document to see if all pages exist in both. Will also check asterisks against revision bars if the document contains the proper tags.

Missing Graphic Check: Checks the document for any missing graphics and flags them (*Only works on XSL-FO printed document.)

Acronym Check: Checks the Acronym Table for any duplicates in either the definition or the acronym.

Foldouts Check: Checks through the document for foldout consistency (I.e, proper pagination, blank pages etc.)

Highlight Check: Uses the highlights to check if document has a revision bar in the location stated by the highlight (*Only works on XSL-FO printed documents).

Full Quality Analysis: This will run all relevant checks for the particular type of document and output a pdf with the flagged comments.

Accuracy Checker:

In order to use the accuracy checker, choose two files you wish to check the accuracy of. These can be PDFS, text, or XML files as long as both files are the same.

When you run the check, you will need to enter the job number and the stage of conversion that the document is at.

Selecting a raw check means that the checker will not attempt to remove or ignore things like extra table headings, blank pages, etc.

PDF Highlighter:

How to use

In order to use the PDF Highlighter, select two PDF files you wish to see the differences between.

When you run the highlighter, you will need to enter the job number.

Active and completed highlight jobs will appear below the highlighter inputs. Depending on the PDF files, highlighting the differences could take seconds or minutes.

The Highlighter will display what stage a give highlight job is on:

  • Parsing PDFs
  • Examining PDFs
  • Generating Comments
  • Highlighting PDFs

Interpreting the results

On the first page, there will be a yellow comment listing how many of each type of highlight is present in the document.

The types of comments and their colours is as follows:

  • Green: High Confidence similar - The highlighted block is considered very similar to another block in the other document.
  • Blue: Medium Confidence similar - The highlighted block is considered reasonably similar to another block in the other document.
  • Purple: Low Confidence similar - The highlighted block is considered similar to another block in the other document.
  • Red: Extra - The highlighted block could not be found in the other document.

Each comment is labeled with a code containing what sort of highlight it is and what number it is in the document.

Beyond Measure

How to use

In order to use Beyond Measure, select a dpf document and press "Output Textblocks".

A text file will be created containing the content of the document:

  • Sorted by block type, ex. Process step, figure title, etc.
  • Without boiler-plate content

Job Folder Creator

How to use

Fill out an excel spreadsheet with the following columns:

  • Job Number
  • D Number
  • Rev Number
  • CAGE Code
  • Classification (USCA/ITAR/UNCLASSIFIED)

For each filled out row, a fob folder structure will be created. For USCA and Unclassified, the folders will be created on their respective drives, but ITAR folders will have to be downloaded from Launchpad once the program has completed.
