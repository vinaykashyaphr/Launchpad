version 0.4.0

Changes To Data Conversion Workflow

The data conversion workflow has been modified so that all activity is done on a single page. Now, when an item is converted, a conversion item will appear with the current status (Pending, In Progress, Completed, or Error), the time until the files are automatically deleted off the server, and a red 'delete' button. Once the conversion finished (status changes to Completed or Error, a green 'download button will appear.')

We've also expanded the conversion tools to be able to handle multiple conversions, downloads, and deletions. You can select multiple source files in the file dialog box, and you can use the checkboxes next to each conversion item to select them and the use the 'Download Selected' or 'Delete Selected' buttons to download or delete all the selected items.

Changes To Arecibo Workflow

Many of the same changes to Data Conversion have also been added to Techwrite, with the exeception that it will still redirect you to a separate page where you can see the script output live. Once Arecibo has completed, a 'Return' button will appear to bring you back to the Tech Writing page, instead of to the Files Ready page.

Added Messages

A message inbox has been added. This will allow you to receive messages from Launchpad. For the time being, this will mostly be for error messages; if something goes wrong during a conversion or Arecibo job, you will receive a message. A copy of this message will also be sent to the admins for debugging.

Added Accuracy Checker

On the Data Conversion, there is now an option to do an accuracy check between two files. Simply select two files and then click on 'Check Accuracy'. Once it's done processing, it will output a percentage similarity between the two files. PDF, XML, TXT files are all permitted. Note that XML comparisons will only compare actual text and not the tags.

Making a Raw Check will do an unbiased comparison between the documents; it will assume that they should be identical. If the 'Raw Check' option is not selected, the program will try and strip out any content that might be unavoidably inconsitent between the two documents. This is intended for comparing and ATA source PDF with an S1000D converted PDF.

Other Changes

  • Added link to QMS website
  • Various UI changes
  • Various stability fixes

Known Issues

  • The client (browser) will sometimes lose connection to the server, resulting in tasks staying at 'Pending' indefinitely. If no input is required, refreshing the page should fix the issue. If input is required, you may need to refresh the page, discard the upload, and try again, as the prompt to enter input may have been missed.
  • Discarding a task in progress may cause the server to crash.
